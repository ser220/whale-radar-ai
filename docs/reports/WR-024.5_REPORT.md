## Design Decisions

- Frozen dataclasses keep the domain layer independent from Pydantic and infrastructure.
- Python 3.9-compatible typing is used throughout.
- Mutable input collections are converted defensively; metadata is recursively frozen.
- Registry order follows deterministic dictionary insertion order; replacement retains position.

## Architecture Deviations

No deviations from the approved WR-024.5 architecture. `app/intelligence/__init__.py`
adds a convenience re-export without integrating the contracts into production.

## Validation Behaviour

Percentage-like numeric fields accept finite real values from 0 through 100,
inclusive. Booleans, non-numeric values, NaN, infinity, and out-of-range values
are rejected. Required text fields reject empty values. Naive timestamps are
rejected; aware timestamps are normalized to UTC.

## Serialization Behaviour

Each model supports `to_dict()` and `from_dict()`. Enum values serialize as
their public strings, timestamps serialize as ISO-8601 UTC values, and a
round-trip restores enum types, immutable collections, and aware UTC datetimes.

## Dependency Audit

`app/intelligence/contracts` imports only Python standard-library modules and
other modules inside the contracts package. It does not import Telegram,
FastAPI, persistence, repositories, the production pipeline, Arkham, source
adapters, funding adapters, or networking clients.
