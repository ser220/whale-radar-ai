# WR-024.5 — Intelligence Contracts Specification

## Objective

Establish a pure, immutable domain-contract package that becomes the common
language for future Whale Radar AI Experts and `MarketStateEngine` without
changing the current production pipeline.

## Scope

- Create `app/intelligence/contracts/`.
- Define the approved enums and immutable contract models.
- Validate percentage-like values and UTC timestamps.
- Provide dictionary serialization and deserialization.
- Provide a lightweight in-memory Expert registry.
- Export the public API through package `__init__.py` files.
- Document and unit-test the contracts on Python 3.9.6.

## Out of scope

- Production pipeline integration.
- WR-025 `MarketStateEngine` implementation.
- Expert implementations or execution orchestration.
- Changes to Arkham ingestion, filtering, scoring, clustering, persistence, or
  Telegram delivery.
- Pydantic schemas, API endpoints, database storage, networking, or dependency
  injection.
- A requirement that probability values sum to 100.

## Required public API

### Enums

- `Direction`: `BULLISH`, `BEARISH`, `NEUTRAL`
- `TrendState`: `TREND`, `CORRECTION`, `REVERSAL`, `RANGE`, `UNKNOWN`
- `LifecycleState`: `WATCH`, `PREPARE`, `READY`, `ACTION`, `MANAGE`, `EXIT`

### ExpertOpinion

- `expert_name: str`
- `direction: Direction`
- `state: str`
- `score: float`
- `confidence: float`
- `quality: float`
- `reasons: tuple/list[str]`
- `warnings: tuple/list[str]`
- `metadata: mapping`
- `timestamp: timezone-aware UTC datetime`

### MarketState

- `trend: TrendState`
- `direction: Direction`
- `strength: float`
- `continuation_probability: float`
- `correction_probability: float`
- `reversal_probability: float`
- `market_maturity: float`
- `decision_stability: float`
- `overall_confidence: float`
- `reasons: tuple/list[str]`
- `warnings: tuple/list[str]`
- `timestamp: timezone-aware UTC datetime`

### DecisionState

- `stage: LifecycleState`
- `trade_readiness: float`
- `eta: optional str`
- `risk: str`
- `confidence: float`
- `action: str`
- `missing_confirmations: tuple/list[str]`
- `timestamp: timezone-aware UTC datetime`

### ExpertRegistry

- `register(expert, replace=False)`
- `get(expert_name)`
- `get_all()`
- `contains(expert_name)`

## Validation rules

- Score, confidence, quality, strength, readiness, maturity, stability, and
  probability fields accept finite real values from 0 through 100 inclusive.
- Booleans, NaN, infinity, non-numeric values, and out-of-range values are
  rejected.
- Required string identifiers and decision fields must not be empty.
- Reasons, warnings, and missing confirmations must contain strings.
- Naive timestamps are rejected.
- Aware timestamps are normalized to `datetime.timezone.utc`.
- Mutable input collections are copied into immutable internal collections.
- Standard nested mappings and collections in metadata are recursively frozen.

## Serialization rules

- Every contract exposes `to_dict()` and `from_dict()`.
- Enums serialize to their public string values and restore to enum instances.
- Timestamps serialize as ISO-8601 strings and restore to aware UTC datetimes.
- `Z` timestamps are accepted during deserialization.
- Tuple-like fields serialize as lists suitable for JSON encoders.
- Metadata serializes to ordinary dictionaries and lists.
- A model round-trip must preserve its semantic values and equality.

## Registry behavior

- Registered objects expose a non-empty `expert_name` attribute.
- Names are unique by default.
- Duplicate registration raises `ValueError`.
- `replace=True` explicitly replaces an existing object.
- Registration order is deterministic.
- Replacing an object retains its original order position.
- Unknown names passed to `get()` use normal mapping behavior and raise
  `KeyError`.

## Forbidden dependencies

The contracts package must not depend on Telegram, FastAPI, Pydantic, database
or repository modules, persistence, the production pipeline, Arkham, funding or
other source adapters, networking clients, or delivery infrastructure.

## Acceptance criteria

- All required files and public exports exist.
- The enums contain exactly the approved values.
- Contract models are frozen dataclasses.
- Validation rules are enforced at construction and deserialization.
- Mutable input collections cannot mutate created contracts.
- Serialization round-trips preserve enums and UTC timestamps.
- Registry behavior, duplicate rejection, replacement, and ordering are tested.
- All created or modified Python files compile under Python 3.9.6.
- Focused `unittest` tests pass.
- Safe existing smoke tests pass.
- Forbidden-import audit passes.
- No production pipeline file is modified.
- `docs/chat_archive/` is not staged or committed.
