# PS-4 Immutable Decision Reference Envelope Contract

## Objective

Define an immutable, verifiable identity envelope for one historical
compatibility decision revision without changing or embedding the existing
compatibility decision contract.

## Public contract

`RealityGapCompatibilityDecisionReference` is a frozen dataclass containing:

- `decision_id`
- `decision_version`
- optional `decision_digest`
- `reference_contract_version`

`decision_version` is always a positive integer and is never inferred from the
digest. When supplied, `decision_digest` must use canonical
`sha256:<64 lowercase hex>` syntax. Version and digest remain independent
historical locators.

## Identity-only boundary

The envelope identifies a historical decision. It does not contain or recreate:

- compatibility status;
- decision basis;
- evaluator output;
- policy logic;
- the decision payload itself.

It performs no compatibility calculation, resolution, lookup, or substitution
of current values. The existing `RealityGapAttachmentCompatibilityDecision`
implementation and serialized shape remain unchanged.

## Historical provenance

The identity plus mandatory version gives association contracts a stable target
for a decision revision. The optional digest can preserve additional snapshot
integrity when an external owner supplies it. No digest is generated by this
contract.

This envelope resolves only the Decision-reference portion of
`GOV-CLOSURE-001`; a corresponding Basis reference and a versioned association
boundary remain separate tasks.

## Validation and serialization

The contract rejects empty identities, booleans or non-positive decision
versions, malformed digests, invalid contract versions, and unknown or missing
serialized fields.

`to_dict()`, `from_dict()`, and `canonical_json()` are deterministic and retain
the explicit `None` digest for version-only references.

## Out of scope

- Resolver or evaluator
- Service or API
- Database or persistence
- Runtime integration or orchestration
- Migration logic
- Modification of Decision, Basis, or Association contracts

## Dependencies

Python standard library and local immutable validation helpers only. No new
third-party dependency is introduced.
