# PS-4 Immutable Basis Reference Envelope Contract

## Objective

Define an immutable, verifiable identity envelope for one historical
compatibility decision-basis revision without changing or embedding the
existing basis contract.

## Public contract

`RealityGapCompatibilityDecisionBasisReference` is a frozen dataclass containing:

- `basis_id`
- `basis_version`
- optional `basis_digest`
- `reference_contract_version`

`basis_version` is always a positive integer and is never inferred from the
digest. When supplied, `basis_digest` must use canonical
`sha256:<64 lowercase hex>` syntax. Version and digest remain independent
historical locators.

## Identity-only boundary

The envelope identifies a historical decision-basis revision. It does not
contain or recreate:

- basis type;
- rule code;
- comparison references;
- evaluator logic;
- the basis payload itself.

Those values remain owned by `RealityGapCompatibilityDecisionBasis`. The
envelope performs no compatibility calculation, resolution, lookup, inference,
or substitution of current values. The existing Basis, Decision, Decision
Reference, and Association implementations and serialized shapes remain
unchanged.

## Historical provenance

The identity plus mandatory version gives association contracts a stable target
for a basis revision. The optional digest can preserve additional snapshot
integrity when an external owner supplies it. No version or digest is inferred
or generated by this contract.

This envelope resolves the Basis-reference portion of `GOV-CLOSURE-001`. A
future versioned association must consume both typed envelopes before closure
can be reconsidered.

## Validation and serialization

The contract rejects empty identities, booleans or non-positive basis versions,
malformed digests, invalid contract versions, and unknown or missing serialized
fields.

`to_dict()`, `from_dict()`, and `canonical_json()` are deterministic and retain
the explicit `None` digest for version-only references.

## Out of scope

- Resolver or evaluator
- Service or API
- Database or persistence
- Runtime integration or orchestration
- Migration logic
- Modification of Basis, Decision, Decision Reference, or Association contracts

## Dependencies

Python standard library and local immutable validation helpers only. No new
third-party dependency is introduced.
